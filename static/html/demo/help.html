This server has been configured to use example data from the OptiPass User Manual. The example is based on six barriers, named A through F. They have been placed on an imaginary map which has two rivers, named Trident and Red Fork.

Example 4

To reproduce the output for Example 4 in the OptiPass manual select the Start tab to show the graphical user interface. You will see controls that let you specify the optimization parameters. Use those controls to make the following settings:

  1. Under Geographic Regions click next to both river names. The barriers on those rivers will appear on the map when a river name is selected. If you move your mouse over a barrier on the map you will see information about that barrier.
  2. Under Budgets click Advanced, drag the slider named Maximum Budget until it shows $500,000, and then click the down arrow next to "#Budgets" until it shows 5.
  3. Under Targets click Weighted and then type 3 next to Target 1 and 1 next to Target 2.

When all the settings have been entered click the button named Run Optimizer. A window will pop up to let you review the settings. If they look correct click Continue (or if you need to change anything click Cancel, make the changes, and click Run again).

Expected Output

This table summarizes the results returned by the OptiPass server:

We told the server to run the optimizer six times: once to compute the available habitat under the current conditions (i.e. with a budget of $0, which will select no barriers), and then with five budgets ranging from $100,000 to $500,000.

Row 4 in the table, for a budget of $400,000, corresponds to the output described in Box 11 of the OptiPass manual. For that budget, the optimal solution consists of barriers A and B. Improving barrier A leads to an increase of 5.285 miles of accessible habitat for target T1 and 5.229 miles for target T2. Since we told OptiPass to assign a weight of 3 for Target 1, the final result in the "WPH" (weighted potential habitat) column is 32.93.

Review the Results

Click the Output tab to see a visual representation of the results.

The graph at the top of the page shows the net gain we can expect at each budget level. The net gain is the increase in WPH over the starting value shown in row 0 of the table.

The table in the Budget Summary section is a condensed version of the table shown above. If you click on a row in the table the map will be updated to show the barriers that were chosen to be in the solution for that row.

The Barrier Details table at the bottom of the page has one row for each barrier. It shows how much it costs to replace a barrier. A green check mark in a column means the barrier was selected as part of the optimal solution for the budget listed at the top of a column.
