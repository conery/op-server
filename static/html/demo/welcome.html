Migratory fish may encounter several man-made barriers during their journey. Tide gates, levees, culverts, and other infrastructure makes it difficult for fish to access vital habitat for foraging, spawning, and resting.

Aging barriers often need to be updated or replaced. Since improving barriers is very costly, stakeholders need to decide how to allocate funds in a way that maximizes improvements. OptiPass™, an application developed by Dr. Jesse O'Hanley of Ecotelligence® LLC, is a software tool that models the important attributes of barriers -- location, cost to repair, and expected benefits after repair -- and idenfifies the optimal set of barriers for any specified budget.

OptiPass runs only on Microsoft Windows® systems. In order to make it easier to deploy applications that want to use OptiPass a project led by The Nature Conservancy has developed two web applications:

  • A server, named op-server, is a RESTful web service that provides an interface to OptiPass.
  • A client, named op-client, is a web application that displays an interface that allows users to select runtime options (select barriers, specify budget levels, etc.) and then pass those optioms to the server to have it run OptiPass.

What are looking at now is an instance of the client. It is configured to connect to a server that has example data from the OptiPass User Manual.

Click the Help tab above to learn how the interface is organized and how to define parameters for an optimization.

When you are ready to run the optimizer click on the Start tab.

When the OptiPass server returns the results from an optimization they can be viewed in the Output tab. The Download tab will have options for saving results.

Acknowledgments

To learn more about OptiPass see OptiPass: The Migratory Fish Passage Optimization Tool, Version 1.1.2 User Manual (O'Hanley 2017), at https://www.ecotelligence.net/.

op-server is written in Python, using the FastAPI library. The code is available on GitHub at https://github.com/conery/op-server. The documentation at https://conery.github.io/op-server explains how to install the server (which must be run on a Microsoft Windows host) and configure it to serve data from a real river system.

op-client is also written in Python, using the Panel library to present the GUI. The code is available on GitHub at https://github.com/conery/op-client. The documentation at https://conery.github.io/op-client explains how to install and configure the server.
